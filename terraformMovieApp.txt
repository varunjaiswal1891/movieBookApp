everytime you rotate IAM access key we need to setup aws configure on termainal
aws configure
AWS Access Key ID	Paste the new access key ID
AWS Secret Access Key	Paste the new secret key
Default region name	e.g. us-east-1
Default output format	json (or leave empty for default)

aws sts get-caller-identity
You should see your account and IAM user/role without errors

check aws configure in terminal
aws configure list 2>&1; echo "---"; aws sts get-caller-identity 2>&1

aws configure list          # Where each setting comes from
aws sts get-caller-identity # Verifies keys work and shows account/ARN

On EC2 verify DB connectivity before restarting:
source /opt/moviebooking/env
nc -zv $DB_HOST 3306
echo "DB credentials set: $([ -n \"$DB_PASSWORD\" ] && echo 'yes' || echo 'no')"

On EC2, restart the app:
# Kill existing process
sudo pkill -f "java -jar" || true

# Download the new JAR
cd /opt/moviebooking
source /opt/moviebooking/env
aws s3 cp s3://$(grep aws s3 cp /opt/moviebooking/start.sh | sed "s/.*s3:\/\/\([^\/]*\)\/.*/\1/")/movie-booking-backend.jar app.jar --region $AWS_REGION

# Or simply run start.sh (it downloads fresh)
sudo /opt/moviebooking/start.sh

# Watch logs
sudo tail -f /var/log/moviebooking.log
